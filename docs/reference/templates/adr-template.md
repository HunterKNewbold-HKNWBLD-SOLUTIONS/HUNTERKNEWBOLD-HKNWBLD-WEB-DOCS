# ADR Template (Architecture Decision Record)

## Title
Short and explicit.

## Status
Proposed / Accepted / Deprecated

## Context
What is true about the system right now. Constraints matter.

## Decision
What we chose and why.

## Alternatives considered
List viable alternatives and why they were rejected.

## Consequences
- Positive
- Negative
- Risks introduced

## Validation
How we will verify this decision is working.

## Rollback
What would trigger reversal and how we would unwind it.

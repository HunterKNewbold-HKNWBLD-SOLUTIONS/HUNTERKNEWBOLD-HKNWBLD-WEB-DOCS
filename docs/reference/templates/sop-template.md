# SOP Template

## Objective
What this procedure accomplishes in one sentence.

## Scope
What is included.

## Out-of-scope
What is explicitly not included.

## Preconditions
What must be true before starting.

## Inputs
Accounts, assets, data formats, access requirements.

## Procedure
Numbered steps. One action per step.

## Validation checks
How to verify success.

## Failure modes
- What breaks
- How you detect it
- Impact
- First response

## Rollback
How to undo changes safely.

## Ownership / Handoff artifacts
What the client receives and who owns it.
